## 📝 第一章创作步骤

### 步骤1：明确本章目标

**本章要完成的3件事**：
1. 介绍主角林浅
2. 系统觉醒（核心事件）
3. 发现丈夫第一个谎言

### 步骤2：设计开头冲突（200-300字）

**从一个冲突开始**：
- 选项1：工作危机（处理公关案件）
- 选项2：意外事故（车祸）✅ 示例采用
- 选项3：发现异常（丈夫行为）

**记住**：第一句话就要吸引人！

### 步骤3：设计中间反转（1,200-1,400字）

**需要包含**：
1. 系统觉醒的过程
2. 第一次使用系统
3. 发现谎言的震惊
4. 2-3次系统弹窗

**反转点**：丈夫温柔的外表下隐藏着谎言

### 步骤4：设计结尾钩子（300-400字）

**留下悬念**：
- 提出问题：车祸是意外还是蓄意？
- 引发任务：系统发布主线任务
- 制造期待：下一章会如何调查？

### 步骤5：添加系统弹窗（3-5次）

**在关键时刻插入**：
1. 系统启动（开头）
2. 首次检测谎言（中间）
3. 情绪分析（中间）
4. 任务发布（结尾）
5. 主线任务（结尾）

## 🎨 创作小技巧

### 技巧1：写作前先列大纲

```
第X章大纲：
1. 开头：[一句话描述冲突]
2. 中间：[列出2-3个场景]
3. 结尾：[一句话描述钩子]
4. 系统：[标记弹窗位置]
```

### 技巧2：对话要有信息量

**每句对话都要**：
- 揭示信息，或
- 推动情节，或
- 展现性格

**避免废话**：
```
❌ "你好啊" "你好" "天气不错" "是啊"
✅ "项目进展如何？" "遇到了问题。"
```

### 技巧3：系统弹窗在转折点

**最佳位置**：
- 人物说完关键话后
- 发现重要线索时
- 情节即将转折前
- 章节高潮时刻

### 技巧4：结尾必留钩子

**五种钩子方式**：
1. 新危机出现
2. 意外访客
3. 重要发现
4. 关键对话
5. 系统警报

## 📋 写作流程推荐

### 第一遍：快速写完（不修改）

1. 开头：200-300字
2. 中间：1200-1400字
3. 结尾：300-400字
4. **不要停下来修改！**

### 第二遍：调整结构

1. 检查三段式是否完整
2. 调整字数到1800-2000
3. 确保有反转
4. 确认有钩子

### 第三遍：优化细节

1. 添加/调整系统弹窗
2. 优化对话
3. 删除注水内容
4. 检查语句通顺

### 第四遍：最终检查

- [ ] 字数1800-2000 ✓
- [ ] 开头有冲突 ✓
- [ ] 中间有反转 ✓
- [ ] 结尾有钩子 ✓
- [ ] 系统弹窗3-5次 ✓
- [ ] 无注水内容 ✓

## 🔍 常见问题速查

### Q：写不出开头怎么办？

**A：用这5个万能开头**：
1. "电话响了。"（坏消息）
2. "门被推开..."（意外访客）
3. 【系统警报】（危机）
4. "你在骗我。"（冲突）
5. "林浅发现了..."（发现）

### Q：中间情节太平淡？

**A：加入这些元素**：
1. 意外发现
2. 关系反转
3. 系统提示新线索
4. 人物秘密暴露
5. 计划被打乱

### Q：结尾想不出悬念？

**A：试试这些**：
1. 在门口停住："门外站着..."
2. 电话响起："是那个号码。"
3. 系统警报：【危险等级：★★★★★】
4. 发现线索："原来如此..."
5. 人物出场："好久不见，林浅。"
